# 📦 Методы Pocket

---

## `pocket/balance`

**💬 Ответ:**
```json
{
  "result": {
    "gold": integer,
    "sweets": float,
    "donate_score": integer,
    "tgstars": integer
  }
}
```

## `pocket/sweets/give`

**📤 Структура запроса:**

| Параметр | Тип   | Описание                                                                               |
|-----------|-------|----------------------------------------------------------------------------------------|
| `user_id` | integer | ✅ ID пользователя, которому отправляются ириски                                        |
| `amount` | float | ✅ Количество ирисок для отправки                                                       |
| `comment` | string | 💬 Комментарий к переводу                                                              |
| `without_donate_score` | boolean | ✅ Использовать донат очки к переводу                                                   |
| `donate_score` | int   | ✅ Максимальное количество очков доната, которые будут использоваться в передаче ирисок |

**💬 Ответ:**
```json
{ "result": integer }
```

## `pocket/sweets/history`

**📤 Структура запроса:**

| Параметр | Тип | Описание |
|-----------|-----|-----------|
| `offset` | integer | 💬 Вернуть записи, где `id ≥ offset` |
| `limit` | integer | 💬 Количество записей в ответе |

**💬 Ответ:**
```json
{
  "result": [
    {
      "date": integer,
      "amount": float,
      "balance": float,
      "to_user_id": integer,
      "details": {
        "total": float,
        "amount": float,
        "fee": float,
        "donate_score": integer
      },
      "comment": string,
      "id": integer,
      "type": string,
      "peer_id": integer
    }
  ]
}
```
`type` может быть `send`, `receive`, `trade`, `dividends`, `trade_sale_order`, `send_with`, `purchase_tgstars` (ириски потрачены на конвертацию в тг-звёзды).

# 🔄 Методы Updates

---

## `updates/getUpdates`

**📤 Структура запроса:**

| Параметр | Тип     | Описание                                                                             |
| -------- | ------- | ------------------------------------------------------------------------------------ |
| `offset` | integer | ID последнего полученного события. Следующие события будут возвращены с `id > offset`. |
| `limit` | integer | Количество записей в ответе |

**💬 Ответ:**
```json
{
  "result": [
    {
      "id": 11,
      "type": "donate_score_log",
      "date": 1756286741,
      "object": { ... }
    }
  ]
}
```
📘 Описание полей:

| Поле     |   Тип   | Описание                                                                                                                        |
| :------- | :-----: | :------------------------------------------------------------------------------------------------------------------------------ |
| `id`     | integer | Уникальный идентификатор события                                                                                                |
| `type`   |  string | Тип события: `sweets_log`, `gold_log`, `donate_score_log`, `tgstars_log`                                                       |
| `date`   | integer | Время события в формате UNIX (секунды)                                                                                          |
| `object` |  object | Объект события. Совпадает по структуре с методами `pocket/sweets/history`, `pocket/gold/history`, `pocket/donate_score/history`, `pocket/tgstars/history` |

# 💰 Методы Trade

---

## `trade/buy`

**📤 Структура запроса:**

| Параметр | Тип | Описание |
|-----------|-----|-----------|
| `price` | float | ✅ Цена, которую вы готовы заплатить за 1 ед. голды |
| `volume` | integer | ✅ Количество голды для покупки |

**💬 Ответ (если сделка совершена):**
```json
{
  "result": {
    "done_volume": integer,
    "sweets_spent": float
  }
}
```

**💬 Ответ (если нет предложений — создаётся заявка):**
```json
{
  "result": {
    "done_volume": integer,
    "sweets_spent": float,
    "new_order": {
      "volume": integer,
      "price": float,
      "id": integer
    }
  }
}
```

---

## `trade/deals`

**📤 Структура запроса:**

| Параметр | Тип | Описание |
|----------|-----|-----------|
| `id` | integer | ID сделки, начиная с которой нужно вернуть данные (по умолчанию 0 — последние сделки) |
| `limit` | integer | Максимум записей (0–200, по умолчанию 200) |

**💬 Ответ:**
```json
{
  "result": [
    {
      "id": integer,
      "group_id": integer,
      "date": integer,
      "price": float,
      "volume": integer,
      "type": string
    }
  ]
}
```

**📘 Описание полей:**

| Поле | Тип | Описание |
|:------|:------:|:------|
| `id` | integer | Уникальный идентификатор сделки |
| `group_id` | integer | Идентификатор группы связанных сделок (0, если сделка одиночная) |
| `date` | integer | Время сделки в формате UNIX (секунды) |
| `price` | float | Цена, по которой была совершена сделка |
| `volume` | integer | Объём сделки |
| `type` | string | Тип сделки: `buy` — покупка, `sell` — продажа |

#  🔗Дип - линки

## 📌 Общая информация о дип-линках

Дип-линки в Ирисе — это специальные Telegram-ссылки, которые позволяют вашему боту инициировать действия через Ирис с обязательным подтверждением со стороны пользователя.

Дип-линки используются для:
- выставление счёта (ириски, голда, очки-доанат, tg-звёзды);
- запроса разрешений на доступ к пользовательской информации.

Ссылка формируется вашим ботом и отправляется пользователю.  
При переходе по дип-линку пользователь попадает в диалог с Ирисом, где видит описание запрашиваемого действия и должен **явно подтвердить его**.

---

### 🔒 Безопасность

- Пользователь **всегда видит**, какое действие выполняется
- Пользователь **сам принимает решение** о подтверждении
- Бот не получает доступ к средствам или данным без согласия пользователя

---

### 👤 Типовой сценарий использования

1. Ваш бот формирует дип-линк с параметрами операции
2. Бот отправляет ссылку пользователю
3. Пользователь переходит по ссылке
4. Ирис показывает описание операции
5. Пользователь нажимает кнопку **«Подтвердить»**
6. Действие выполняется (перевод / выдача прав)

# ⚠️ Ошибки API

Формат ошибки (`ApiError`):
```json
{
  "error": {
    "code": 401,
    "description": "Invalid token"
  }
}
```

Пример с кодом 400:
```json
{
  "error": {
    "code": 400,
    "description": "Required parameter \"{parameter}\" is absent"
  }
}
```

**Коды и сообщения:**

| Код | Сообщение | Когда возвращается |
|:---:|:----------|:-------------------|
| 400 | Required parameter "{parameter}" is absent | Не указан обязательный параметр |
| 400 | User is 0 or not a number | Некорректный `user_id` |
| 400 | User denied receiving | Переводы указанному пользователю запрещены |
| 400 | Длина текста слишком длинная | `comment` > 128 символов |
| 400 | Ссылки на сторонние сайты в тексте запрещены | В тексте обнаружены внешние ссылки |
| 400 | Обращения к пользователям в тексте запрещены | Обнаружены обращения к пользователям |
| 400 | Слишком много смайлов в тексте | >99 эмодзи в тексте |
| 400 | Price is 0, negative or not a number | Неверно указана цена |
| 400 | Volume is 0, negative or not a number | Неверно указан объём |
| 400 | ID is 0, negative or not a number | Неверный `id` заявки |
| 400 | Tg stars is 0 or not a number | Неверно указано количество тг-звёзд |
| 400 | Version is wrong | Неверный номер версии в `/v{номер версии}/...` |
| 401 | Invalid token | Токен не принадлежит указанному боту |
| 403 | Rights are not given | Пользователь не дал разрешение на просмотр |
| 404 | Handler Not Found | Метод не существует или указан с ошибкой |
| 404 | User not found | Пользователь не найден |
| 404 | Order not found | Заявка по `id` не найдена |
| 404 | No such bot | ID не принадлежит боту |
| 409 | Not enough {currently}. Need {quantity} | Недостаточно средств/ресурсов |
| 429 | Flood control | Более 3 запросов за секунду |

# ❗Объявленные правила и рекомендации по использованию API Ириса

## ❗ **`Логирование операций цифровых активов`**
>
> Важным условием предоставления Iris Connect API является согласие предоставить по запросу администрации Ириса логи переводов активов запрашиваемого пользователя.
>
> 🕵️ Это требование связано с расследованием скам-цепочек, в которых мошенники могут использовать вашего бота для отмывания цифровых активов.
>
> **Под цифровыми активами понимаются:**
> - ириски, вводимые в бота и выводимые из бота;
> - внутренняя валюта вашего бота;
> - предметы, скины, бонусы и другие игровые активы.
>
> **Минимальный набор логов:**
> 1. Кто ввёл ириски в бота, когда и в каком количестве
> 2. Кому были переданы активы внутри бота
> 3. Кто и когда вывел ириски из бота

## ❗ **`Не используйте «бесконечные» версии API`**
>
> Указание условной «вечной последней версии» (например, `/v100`) является плохой практикой и может привести к **непредсказуемым поломкам** при изменениях в API.
> Использование несуществующих или «будущих» версий приводит к тому, что клиенты API могут внезапно перестать работать в неожиданных местах.
> 
> В связи с этим пользователям **настоятельно рекомендуется указывать только реально существующие версии API**.
>
> При указании несуществующей версии API будет возвращена ошибка.
>
> ### 🔎 Проверка актуальной версии
>
> Для получения текущей актуальной версии Iris Connect API доступен метод: [/last_version](#last_version)
